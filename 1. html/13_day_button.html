Form-Related Attributes:

formaction — where to send data (overrides form's action)
formmethod — get or post (overrides form's method)
formtarget — where to open response (_blank, _self, etc.)
formnovalidate — skips form validation for this submit

type attribute:

submit — default type inside a <form> sends form data
reset — clears all form fields back to default
button — does nothing unless you add JavaScript
button type
Does absolutely nothing by itself. It's a plain clickable button meant to be controlled with JavaScript.

Without onclick (or JS), it just sits there and does nothing when clicked — no form submission, no reset.
submit type
Sends the form data to the server. This is the default type if you don't specify one inside a form.
reset type
Clears all form fields back to their original/default values.

common attributes:

id — unique identifier, used to target with CSS/JS
class — for CSS styling
name — name sent with form data (used with value)
value — value sent to server when button submits form
disabled — greys out button, makes it unclickable
autofocus — automatically focuses button when page loads
form — links button to a form by its id, even if button is outside the <form> tags
title — tooltip text shown on hover
