Inline vs Block Elements in HTML

Block-level elements

  • Start on a new line, take up the full width available (by default)
  • Can contain other block or inline elements
  • Respect width, height, margin, and padding on all sides

Common examples

<div>, <p>, <h1>-<h6>, <ul>, <ol>, <li>,
<section>, <article>, <header>, <footer>,
<form>, <table>, <blockquote>

Inline elements

  • Do not start on a new line — flow within surrounding text
  • Only take up as much width as their content needs
  • Ignore width/height; vertical margin/padding may not affect layout as expected (horizontal padding/margin works fine)
  • Typically can't contain block-level elements

Common examples

<span>, <a>, <strong>, <em>, <img>,
<label>, <input>, <button>, <code>

Quick comparison

Feature Block Inline
New line Yes No
Width Full container width (default) Content width only
Height/Width CSS Respected Ignored
Margin/Padding All sides respected Only left/right reliably affect layout
Nesting Can contain block & inline Generally only inline/text

Example

<div>
  This is a block element containing
  <span>an inline span</span> and
  <strong>bold inline text</strong>.
</div>
<p>This paragraph starts on a new line.</p>

Here, the <span> and <strong> stay inline with the text, but <div> and <p> each start on their own line.

CSS display property (reference only, not applied here)

You can override default behavior:

span { display: block; }   /* now behaves like a block element */
div  { display: inline; }  /* now behaves like an inline element */

There's also inline-block, which flows inline like a normal inline element but does respect width, height, and full margin/padding — useful for things like nav menu items or buttons. 

.nav-item { display: inline-block; width: 100px; padding: 10px; }
